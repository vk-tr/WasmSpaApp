# WasmSpaApp

Single page приложение,
написанное с помощью технологии web assembly
и фреймворка blazor

### Уточнения по проекту
* В качестве хранилища используется localStorage, разделенный между двумя сервисами, один из которых имитирует поведение стандартной СУБД, а другой служит хранилищем временных объектов(в данном случае хранит информацию об активном пользователе);
* Есть возможность добавлять пользователя через форму входа (вбить данные в поля формы входа и нажать **Sign In**). Т.к. localStorage должен быть инициализирован, при старте приложения создается root пользователь с логпасом root/admin и ролью администратора;
* Администратор видит четвертую вкладку и может удалять пользователей и выдавать(или наоборот) им права администратора. Исключение составляет рутовый пользователь - его изменять нельзя, это сделано с точки зрения сохранения целостности хранилища. Так же нельзя изменять права и удалять текущего пользователя;
* Пароли пользователей хешируются при сохранении в localStorage, что защищает их от просмотра в дев консоли, а при выводе в UI все символы в пароле заменяются на символ '*';
* Аутентификация и авторизация пользователя реализованы через кастомные компоненты, документации по AuthorizeView крайне мало, + класс, который бы должен был стать Wrapp-ом для изменения логики проверки совсем не хотел наследоваться от базового компонента (не находил ссылку на базовый компонент). Аутентификация производится при переходе на каждой странице, авторизация - при загрузке меню навигации;
* Факт успешной аутентификации можно посмоотреть на первой странице - выводится сообщение вида Hello, @username. Факт авторизации - отображение четвертой вкладки в меню навигцаии (Вкладка админ панели);
* Добавлена документация по коду;
* Добавлена базовая обработка ошибок при обращении к хранилищу, имитирующему поведение СУБД.
